# payment-system

基于微服务的支付系统设计与实现。

## 1. 项目简介

本项目面向支付业务场景，基于微服务架构拆分账户、交易、对账、用户等核心模块，围绕**转账主流程、消息驱动对账、异常状态流转与补偿机制**展开设计与实现。项目主要用于训练 Java 后端工程能力，并为后续简历投递、毕业设计以及性能优化研究提供实践基础。

当前项目已完成一条可验证的业务闭环：

**转账请求 → 交易单创建 → 账户扣款/入账 → 账户流水记录 → RocketMQ 发送成功消息 → 对账服务消费消息 → 对账记录生成 → 对账成功**

## 3. 模块划分

### 3.1 payment-account
账户模块，负责账户信息查询、余额变更、扣款、入账、账户流水记录等能力。

### 3.2 payment-trade
交易模块，负责转账主流程编排、交易记录生成、交易状态流转、交易成功消息发送等能力。

### 3.3 payment-reconcile
对账模块，负责消费交易成功消息、生成对账记录、查询统计、异常处理与补偿任务。

### 3.4 payment-user
用户模块，提供用户相关基础能力。

### 3.5 payment-common
公共模块，封装通用返回结构、DTO 等基础组件。

## 4. 当前已实现功能

### 4.1 微服务基础拆分
- 完成账户、交易、对账、用户、公共模块的项目拆分
- 支持各模块独立启动与协作

### 4.2 转账主流程
- 支持转账请求发起

- 支持转入账户入账
- 支持交易记录落库

### 4.3 消息驱动对账
- 交易成功后发送 RocketMQ 消息
- 对账模块异步消费交易成功消息
- 自动生成对账记录
- 实现交易模块与对账模块解耦

### 4.4 对账查询能力
- 查询全部对账记录

### 4.8 基础性能验证

## 5. 核心业务流程

### 5.1 转账与对账主链路

1. 用户发起转账请求
2. 交易模块执行转账主流程
3. 账户模块完成扣款与入账
4. 交易模块记录交易结果
5. 交易成功后发送 RocketMQ 消息
6. 对账模块消费消息并生成对账记录
7. 若出现异常，可通过异常标记、恢复、批量补偿、定时补偿完成修复

---

## 6. 对账状态说明

- `0`：待对账
- `1`：对账成功
- `2`：对账异常

当前支持的状态流转：

- `0 -> 1`：正常对账成功
- `1 -> 2`：手动标记异常
- `2 -> 1`：手动恢复
- `2 -> 1`：批量补偿恢复
- `2 -> 1`：定时任务自动补偿恢复

---

## 7. 项目亮点

- 基于 RocketMQ 实现交易成功事件异步通知，完成交易与对账解耦，体现最终一致性处理思路
- 对账模块形成“异常标记—手动恢复—批量补偿—定时补偿”的完整闭环
- 基于状态校验实现基础幂等控制，避免重复操作导致数据状态错乱
- 通过账户流水、交易单、对账记录三类数据形成可追踪的业务链路
- 在本地多微服务环境下完成从转账到对账成功的全链路联调验证
- 已完成基础压测，为后续分页优化、并发测试和性能分析提供依据
